# Contribution Guide

This is an educational term project. We welcome all contributions!

## How to Contribute?

### 1. Report Bugs
- Open a new issue in the GitHub Issues section
- Provide a clear description of the bug
- Include steps to reproduce the issue

### 2. Suggest Features
- Open an issue for new feature suggestions
- Explain why this feature would be useful
- Provide example use cases if possible

### 3. Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## Coding Standards

- Use C++11 or higher
- Use clear and English variable names
- Add comments to explain functions and classes
- Ensure code compiles without errors
- Keep code style consistent with existing code

## Commit Messages

A good commit message should follow this format:
```
Short description (less than 50 characters)

More detailed explanation if needed.
- Change made 1
- Change made 2
```

## Pull Request Process

1. Test your branch thoroughly before opening a PR
2. Describe what you did in the PR description
3. Reference related issues in your PR
4. Be open to feedback

## Questions?

Use the GitHub Discussions section to ask questions.
